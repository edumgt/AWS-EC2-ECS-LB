# EC2 실습 03 - ASG 운용 점검과 부하 테스트

## 목표
- Launch Template 기반으로 ASG를 운용하고, 상태 점검 루틴을 익힙니다.
- SSH 접속 이슈, 서비스 비정상 상태를 빠르게 복구합니다.

## 1) 인스턴스 접속 점검
```bash
ssh -i "<KEY_NAME>.pem" ubuntu@<PUBLIC_IP>
```

접속 불가 시 확인:
1. SG 인바운드 22 허용
2. 퍼블릭 IP/라우팅/IGW
3. 키 파일 권한 (`chmod 400`)

### EC2 Instance Connect 사용 시 추가 확인
AWS 콘솔에서 `Failed to connect to your instance` 또는 `Access denied by EC2 Instance Connect`가 나오면, 단순히 22번 포트만 열려 있어도 접속이 되지 않을 수 있습니다.

우선 점검:
1. 인스턴스에 퍼블릭 IPv4 또는 EIP가 연결되어 있는지 확인
2. 퍼블릭 서브넷 라우팅 테이블에 `0.0.0.0/0 -> Internet Gateway`가 있는지 확인
3. 보안 그룹 인바운드에 `TCP 22`가 열려 있는지 확인
4. 접속 사용자에 `ec2-instance-connect:SendSSHPublicKey` 권한이 있는지 확인
5. AMI/OS가 EC2 Instance Connect를 지원하는지 확인

참고:
- Ubuntu 계열은 보통 사용자명이 `ubuntu`
- Amazon Linux 계열은 보통 사용자명이 `ec2-user`
- 브라우저 기반 `EC2 Instance Connect`가 계속 거절되면, 생성한 키 페어로 일반 SSH 접속을 먼저 검증하는 편이 더 빠를 수 있습니다.

CLI 점검 예시:
```bash
aws ec2 describe-instances \
  --instance-ids <INSTANCE_ID> \
  --query "Reservations[].Instances[].{PublicIp:PublicIpAddress,SubnetId:SubnetId,State:State.Name,Platform:PlatformDetails}" \
  --output table

aws ec2 describe-route-tables \
  --filters Name=association.subnet-id,Values=<SUBNET_ID> \
  --query "RouteTables[].Routes[]"

aws ec2 describe-security-groups \
  --group-ids <SG_ID> \
  --query "SecurityGroups[].IpPermissions"
```

## 2) 웹 서비스 점검
```bash
sudo lsof -i :80
sudo netstat -tulnp | grep ':80'
sudo apt update
sudo apt install nginx -y
sudo systemctl enable --now nginx
```

## 3) ASG 상태 점검
```bash
aws autoscaling describe-auto-scaling-groups --auto-scaling-group-names <ASG_NAME>
aws elbv2 describe-target-health --target-group-arn <TG_ARN>
```

## 4) 부하 테스트용 Launch Template 예시
```bash
aws ec2 create-launch-template \
  --launch-template-name stress-test-template \
  --version-description "v1" \
  --launch-template-data '{
    "ImageId": "ami-xxxxxxxx",
    "InstanceType": "t3.micro",
    "KeyName": "<KEY_NAME>",
    "SecurityGroupIds": ["sg-xxxxxxxx"]
  }'
```
